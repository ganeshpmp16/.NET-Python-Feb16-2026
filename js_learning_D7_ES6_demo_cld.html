🗂 Arrays & Array Methods

Arrays are ordered collections that store multiple values in a single variable. JavaScript arrays are dynamic, zero-indexed, and can hold mixed data types.

1.1 Creating Arrays

// Array literal (recommended)
const fruits = ['apple', 'banana', 'cherry'];

// Array constructor
const nums = new Array(1, 2, 3);

// Mixed types (allowed in JS)
const mixed = [42, 'hello', true, { name: 'Alice' }, [1, 2]];

// Access by index (zero-based)
console.log(fruits[0]);         // 'apple'
console.log(fruits.at(-1));     // 'cherry' (ES2022 .at())
console.log(fruits.length);      // 3

1.3 Non-Mutating / Functional Methods (return new values)

const numbers = [1, 2, 3, 4, 5, 6];

// map() — transform every element into a new array
const doubled = numbers.map(n => n * 2);
// [2, 4, 6, 8, 10, 12]

// filter() — keep elements matching a condition
const evens = numbers.filter(n => n % 2 === 0);
// [2, 4, 6]

// reduce() — collapse array to a single value
const sum = numbers.reduce((acc, n) => acc + n, 0);
// 21

// find() — first element matching condition (or undefined)
const firstBig = numbers.find(n => n > 3);
// 4

// some() / every() — boolean checks
const hasEven = numbers.some(n => n % 2 === 0);   // true
const allPos  = numbers.every(n => n > 0);          // true

// flat() — flatten nested arrays
[1, [2, [3]]].flat(Infinity);   // [1, 2, 3]

// flatMap() — map then flatten (1 level)
['Hello World', 'foo bar'].flatMap(s => s.split(' '));
// ['Hello', 'World', 'foo', 'bar']

// slice() — copy a portion
numbers.slice(1, 4);   // [2, 3, 4] (original unchanged)

// includes() — check membership
numbers.includes(3);   // true

// indexOf() — find position
numbers.indexOf(4);    // 3

1.4 Chaining Methods — Real World Example

Multiple array methods can be chained together for expressive data pipelines:

const students = [
  { name: 'Alice', score: 85, passed: true  },
  { name: 'Bob',   score: 42, passed: false },
  { name: 'Carol', score: 91, passed: true  },
  { name: 'Dave',  score: 67, passed: true  },
  { name: 'Eve',   score: 55, passed: false },
];

// Get names of passing students sorted by score (high→low)
const topStudents = students
  .filter(s => s.passed)
  .sort((a, b) => b.score - a.score)
  .map(s => `${s.name}: ${s.score}`);

// ['Carol: 91', 'Alice: 85', 'Dave: 67']

// Average score of all students
const avg = students
  .reduce((sum, s) => sum + s.score, 0) / students.length;

// 68

⏳ Asynchronous JavaScript

JavaScript is single-threaded, but asynchronous programming lets it handle long operations (network requests, file I/O, timers) without blocking the UI. There are three main patterns.

2.1 The JavaScript Event Loop

The event loop is what makes async JS work. It continuously checks the Call Stack and Callback Queue, moving callbacks onto the stack when it's empty.

// Synchronous — blocks the stack until done
console.log('1 - Start');
console.log('2 - Middle');
console.log('3 - End');   // Output: 1, 2, 3 (in order)

// Asynchronous — setTimeout schedules callback for LATER
console.log('A - Start');
setTimeout(() => console.log('B - Timer fires!'), 0);
console.log('C - End');
// Output: A, C, B  ← B runs AFTER current stack clears

2.2 Callbacks — The Original Async Pattern

A callback is a function passed as an argument to another function, to be called later when an async operation completes.

// Simulating an async operation with a callback
function fetchUser(userId, callback) {
  console.log(`Fetching user ${userId}...`);
  setTimeout(() => {
    const user = { id: userId, name: 'Alice', role: 'Admin' };
    callback(null, user);   // null = no error
  }, 800);
}

fetchUser(1, (err, user) => {
  if (err) { console.error(err); return; }
  console.log(`Got user: ${user.name} (${user.role})`);
});

// ⚠️ Callback Hell — deeply nested callbacks become unreadable
getUser(1, (err, user) => {
  getPosts(user.id, (err, posts) => {
    getComments(posts[0].id, (err, comments) => {
      // Keep going... this is "Callback Hell" 🔥
    });
  });
});

2.3 Promises — Solving Callback Hell

A Promise is an object representing the eventual completion or failure of an async operation. It can be in one of three states: pending, fulfilled, or rejected.

// Creating a Promise
function fetchUser(userId) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (userId > 0) {
        resolve({ id: userId, name: 'Alice' }); // success
      } else {
        reject(new Error('Invalid user ID'));     // failure
      }
    }, 600);
  });
}

// Consuming a Promise with .then() / .catch() / .finally()
fetchUser(1)
  .then(user => {
    console.log(`User: ${user.name}`);
    return fetchUser(2);  // chain another promise!
  })
  .then(user2 => console.log(`User 2: ${user2.name}`))
  .catch(err  => console.error(`Error: ${err.message}`))
  .finally(() => console.log('Done!'));

// Promise.all — run multiple promises IN PARALLEL
Promise.all([fetchUser(1), fetchUser(2)])
  .then(([u1, u2]) => console.log(u1.name, u2.name));

// Promise.allSettled — run all, get all results (even failed)
Promise.allSettled([fetchUser(1), fetchUser(-1)])
  .then(results => results.forEach(r => console.log(r.status)));

2.4 Async / Await — The Modern Way

async/await is syntactic sugar over Promises, making async code look and read like synchronous code — much easier to follow.

// async keyword marks a function as returning a Promise
async function getUserData(userId) {
  try {
    const user  = await fetchUser(userId);    // wait for promise
    const posts = await fetchPosts(user.id); // wait for next
    console.log(`${user.name} has ${posts.length} posts`);
    return { user, posts };
  } catch (err) {
    console.error(`Failed: ${err.message}`);
  } finally {
    console.log('Request finished');
  }
}

getUserData(1);  // call like normal function

// Parallel execution with async/await
async function loadAll() {
  // ❌ Sequential (slower — each waits for previous)
  const u1 = await fetchUser(1);
  const u2 = await fetchUser(2);

  // ✅ Parallel (faster — both fire at once)
  const [u3, u4] = await Promise.all([fetchUser(3), fetchUser(4)]);
}

// Real fetch() API call with async/await
async function getJoke() {
  const response = await fetch('https://official-joke-api.appspot.com/random_joke');
  const joke    = await response.json();
  console.log(`${joke.setup} ... ${joke.punchline}`);
}

ES6 (ECMAScript 2015) and later versions added powerful new syntax and features that make JavaScript more expressive, concise, and easier to work with.

3.3 Template Literals

const name  = 'Alice';
const score = 95;

// Old way — string concatenation
'Hello, ' + name + '! Score: ' + score;

// Template literal — backtick + ${expression}
`Hello, ${name}! Score: ${score}`;

// Any JS expression inside ${}
`${name.toUpperCase()} scored ${score >= 90 ? 'A' : 'B'}`;

// Multi-line strings (no \n needed!)
const html = `
  <div class="card">
    <h2>${name}</h2>
    <p>Score: ${score}</p>
  </div>
`;

// Tagged templates (advanced)
function highlight(strings, ...vals) {
  return strings.reduce((acc, str, i) =>
    acc + str + (vals[i] ? `<b>${vals[i]}</b>` : ''), '');
}
highlight`Name: ${name}, Score: ${score}`;
// "Name: <b>Alice</b>, Score: <b>95</b>"

3.5 Spread & Rest Operators

// ——— SPREAD (...) — expands an iterable ———

// Copy an array (shallow)
const a = [1, 2], b = [...a, 3, 4];   // [1,2,3,4]

// Merge arrays
const merged = [...[1, 2], ...[3, 4]];  // [1,2,3,4]

// Spread into function args
Math.max(...[5, 12, 3]);   // 12

// Copy/merge objects
const base    = { a: 1, b: 2 };
const updated = { ...base, b: 99, c: 3 };
// { a:1, b:99, c:3 }  — later keys win

// ——— REST (...) — collects remaining items ———

function sum(...nums) {
  return nums.reduce((acc, n) => acc + n, 0);
}
sum(1, 2, 3, 4);   // 10

const [head, ...tail] = [10, 20, 30];
// head=10, tail=[20,30]

3.6 Modules (import / export)

// ── math.js ──────────────────────────────

// Named exports — export multiple things
export const PI = 3.14159;

export function add(a, b)    { return a + b; }
export function multiply(a, b) { return a * b; }

// Default export — one per file
export default function square(n) { return n * n; }


// ── app.js ────────────────────────────────

// Import default + named in one statement
import square, { PI, add, multiply } from './math.js';

// Import and rename
import { add as sum } from './math.js';

// Import entire module as namespace
import * as Math from './math.js';
Math.add(2, 3);

// Dynamic import (lazy load)
const { add: addFn } = await import('./math.js');

// In HTML:
// <script type="module" src="app.js"></script>
💡 Modules are scoped by default — variables don't leak to global scope. Always use type="module" on your script tag.

3.7 Other Key ES6+ Features

// ── Optional Chaining (?.) ─────────────────────────────────
const user = { profile: { name: 'Alice' } };
user?.profile?.name;        // 'Alice'
user?.address?.city;        // undefined (no error!)
user?.greet?.();            // undefined if greet doesn't exist

// ── Nullish Coalescing (??) ────────────────────────────────
const count = 0;
count || 10;    // 10 (0 is falsy!)
count ?? 10;    // 0  (?? only triggers on null/undefined)

// ── Shorthand Object Properties ────────────────────────────
const name = 'Alice', age = 30;
const person = { name, age };  // { name: 'Alice', age: 30 }

// ── Computed Property Names ────────────────────────────────
const key = 'score';
const obj  = { [key]: 95 };   // { score: 95 }

// ── Logical Assignment Operators (ES2021) ──────────────────
let a = null;
a ??= 'default';   // assigns 'default' if a is null/undefined

let b = false;
b ||= 'fallback'; // assigns 'fallback' if b is falsy

// ── Structured Clone (ES2022) ──────────────────────────────
const deep = structuredClone({ nested: { value: 42 } });
// True deep copy — unlike spread which is shallow
